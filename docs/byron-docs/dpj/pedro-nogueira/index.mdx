Problema: "Cliente decide mudar algo importante no projeto perto do prazo final"

Resolução: 

- Bem, primeiramente é preciso averiguar dois casos que poderiam acontecer diante da situação:

1. O Cliente não pode estender o prazo de entrega:

<center>

![Cliente insatisfeito](./img/cliente_insatisfeito_resized.jpg)

</center>

Nesse caso, é preciso averiguar o projeto com e sem a mudança, ou seja, avaliar a verdadeira necessidade de implementar a mudança desejada pelo cliente. Caso for uma mudança que não fará muita diferença na usabilidade do projeto apresente os aspectos positivos e negativos que trará e o porquê dessa alteração pode não ser atrativa para o cliente final, tente o convencer que essa não é a melhor opção para o projeto. Se for uma mudança indispensável e que afeta no funcionamento base do projeto, é preciso ter uma transparência com o cliente de que essa alteração impacta a maioria das funcionalidades já existentes, o que implica na reestruturação de grande parte do código. Se for uma muança que é possível dentro do prazo converse com toda equipe para elaborar uma nova divisão de tarefas e organize de forma que não os afete muito.

2. O Cliente pode estender o prazo de entrega:

<center>

![Cliente satisfeito](./img/cliente_satisfeito_resized.png)

</center>
Nesse caso, só é preciso contatar a equipe sobre a nova exigência do cliente e reorganizar o planejamento de entrega com base na nova funcionalidade e no novo prazo pedido.
