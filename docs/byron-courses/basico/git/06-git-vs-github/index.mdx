# Git e Github: qual a diferença?

Essa aqui vai ser bem curta.

Tudo que você viu até agora, foi sobre o Git, a ferramenta de versionamento de
código, o programa instalável que você instalou na segunda aula e tem usado até
agora, pelo terminal ou por meio de GUIs, como o GitKraken.

Isso é o Git.

Github, por outro lado, é uma plataforma na internet, um website, uma aplicação
web, que permite a hospedagem de um **repositório remoto**.

Um repositório remoto nada mais é que um cópia de um repositório local, ou
vice-versa. Lembre-se da característica distribuída do Git.

Por meio de um repositório remoto, desenvolvedores trabalhando em um mesmo
projeto têm um _**single source of truth**_, uma referência do que o projeto é.
Ele é o que está no repositório remoto.

Todos os desenvolvedores que contribuem para a codebase, fazem um _**clone**_ do
repositório remoto. Todos os repositórios locais partem dele nesse caso.

Além disso, o Github funciona como uma "rede social" para programadores, também
possui funcionalidades para automação de tarefas, espaços de trabalho na nuvem,
entre outras coisas. Mas vamos focar só no seu aspecto de **plataforma para
hospedar repositórios remotos**.

Na próxima aula eu vou te mostrar como criar uma conta no Github e também como
utilizar algumas frentes dessa plataforma, que escolhemos para hospedar os
projetos da byron.
