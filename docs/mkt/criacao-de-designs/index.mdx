## Etapas do processo

### Capacitação
**Quem:** designer do projeto.

**Quando:** antes de iniciar o design.

**Por que:** para entender melhor as etapas do processo e conhecer as ferramentas necessárias para a realização deste.

**Como:** fazer capacitação sobre web design. Para isso, fica como sugestão o [curso de Figma: Design visual de um site mobile](https://cursos.alura.com.br/course/figma-design-visual-site-mobile)

### Criação de wireframes

**Quem:** designer do projeto.

**Quando:** assim que o contrato for assinado.

**Por que:** evitar retrabalho na próxima etapa.

**Como:** montagem do esqueleto do produto final, feito com caixas em cores neutras. Usado para mostrar a estrutura do projeto para o cliente visando a compatibilidade de ideias.

<image src={wireframe} alt="Exemplo de wireframe" />

### Criação de protótipos

**Quem:** designer do projeto.

**Quando:** assim que o wireframe for aprovado.

**Por que:** para fazer testes com o cliente e evitar retrabalho na fase final.

**Como:** conversar com o cliente sobre a identidade visual, ou seja, sobre cor, tipografia e imagem. Desse modo, é possível estruturar um pouco melhor o wireframe. O protótipo também deve mostrar como será o funcionamento e o comportamento do site, por exemplo, suas transições.

<image src={prototipo} alt="Exemplo de protótipo" />

### Criação de designs
**Quem:** designer do projeto.

**Quando:** assim que o cliente aceitar o protótipo.

**Por que:** conclusão do trabalho.

**Como:** deve ser refinado o trabalho feito no wireframe e protótipo.
